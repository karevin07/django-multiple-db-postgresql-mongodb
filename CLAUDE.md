## Project Overview

This is a Django project demonstrating multi-database architecture with both PostgreSQL and MongoDB. The project uses uv for dependency management and Docker for containerization.

**Key architectural pattern**: Different Django apps route to different databases:
- `example_user` → PostgreSQL (`default` database)
- `example_app` → MongoDB (`mongo_db` database)
- Django system apps (admin, auth, contenttypes, sessions) → PostgreSQL

## Database Architecture

### Multiple Database Setup

The project uses **two separate settings files**:
- `myproject/settings.py` - Base settings (not used directly)
- `myproject/multiple_db_settings.py` - **Active settings** with multi-database configuration

**CRITICAL**: Always use `--settings=myproject.multiple_db_settings` when running Django commands.

### Database Router

`myproject/routing.py` contains `MyDBRouter` which routes models to appropriate databases based on `DATABASE_APPS_MAPPING`:
```python
DATABASE_APPS_MAPPING = {
    'admin': 'default',
    'auth': 'default',
    'contenttypes': 'default',
    'sessions': 'default',
    'example_user': 'default',      # PostgreSQL
    'example_app': 'mongo_db',      # MongoDB
}
```

### MongoDB Models

Models using MongoDB must:
1. Use `ObjectIdAutoField` for primary key
2. Explicitly set `app_label` in Meta class
3. Be mapped to `mongo_db` in `DATABASE_APPS_MAPPING`

Example from `example_app/models.py:8`:
```python
class Post(models.Model):
    class Meta:
        app_label = 'example_app'

    id = ObjectIdAutoField(primary_key=True)
```

## Development Commands

### Docker Environment (Recommended)

```shell
# Build Django image
make build-django-image

# Start all services (Django, PostgreSQL, MongoDB, pgAdmin, mongo-express)
docker-compose up -d

# Run migrations (must be done in specific order)
make migrate-all

# Create superuser
make create-superuser-local
```

### uv (Local Development)

```shell
# Install dependencies
make uv-sync

# Run Django server locally
cd django_project
uv run python manage.py runserver --settings=myproject.multiple_db_settings
```

### Migrations

**IMPORTANT**: Migrations must be run per-database with explicit `--database` flag:

```shell
# Make migrations
make make-migrate

# Migrate all (runs migrations in correct order)
make migrate-all

# Individual migration commands
make migrate-admin      # admin → default (PostgreSQL)
make migrate-auth       # auth → default
make migrate-user       # example_user → default
make migrate-app        # example_app → mongo_db (MongoDB)
```

### JupyterLab with Django

Run JupyterLab with Django kernel for interactive development:

```shell
make run-jupyter-with-django
# Access at http://0.0.0.0:8888
```

Environment variables are loaded from `.env` file (see `example_env` for template):

## Adding New Apps

When creating apps that should use MongoDB:

1. Add app to `INSTALLED_APPS` in `multiple_db_settings.py`
2. Add mapping to `DATABASE_APPS_MAPPING` in `multiple_db_settings.py`
3. Update `MyDBRouter` label lists in `routing.py` if needed
4. Use `ObjectIdAutoField` for model primary keys
5. Set `app_label` explicitly in model Meta class
6. Run migrations with `--database=mongo_db` flag
